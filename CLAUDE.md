# Slowmail

A fictional email service that intentionally delays sending emails by 24 hours. The concept is a meditation on patience, anticipation, and the lost art of slow communication.

## Project Structure

```
index.html        # Gallery page — showcases all 25 landing page iterations
v1.html           # Landing page iteration 1
v2.html           # Landing page iteration 2
...
v25.html          # Landing page iteration 25
```

## Goals

- Design 25 distinct landing page iterations for Slowmail (v1.html–v25.html)
- Build one gallery page (index.html) that links to and previews all 25 iterations
- Explore a wide range of visual directions, tones, and design approaches across iterations

## Constraints

- **Pure HTML and CSS only** — no JavaScript, no frameworks, no libraries
- No build tools, no preprocessors, no external dependencies
- All styles must be inline `<style>` blocks or in a `<link>` to a local `.css` file
- No web fonts loaded from CDNs (use system font stacks only)

## Design Principles

Each iteration should feel intentional and complete. Across the 25 versions, vary:
- Visual tone (minimal, ornate, playful, serious, retro, futuristic, etc.)
- Layout approach (centered hero, full-bleed, asymmetric, grid-based, etc.)
- Color palette
- Typography choices (using system fonts)
- Copy angle (aspirational, humorous, philosophical, utilitarian, etc.)

## The Concept

Slowmail delays every email by exactly 24 hours. Landing pages should communicate this core value proposition clearly, even as the visual treatment changes dramatically between iterations.
